Granula

is a fine-grained performance evaluation framework for Big Data Processing (BDP) systems. Granula facilitates in-depth performance analysis of BDP jobs by establishing an efficient way to explore the internals of BDP systems.

Granula

consists of three main modules: modeller, archiver and visualizer. Performance evaluation within the Granula framework is incremental and continuously-evolving, and the results are verifiable and highly reusable.

Granula Modeller

The modelling module of the Granula framework allows performance analysts to build performance models incrementally for Big Data Processing (BDP) jobs. The performance models instruct how performance information should be logged in BDP systems, and define how performance information in these logs can be extracted and converted into a performance archive. The modelling process is reusable and continuously-evolving.

Granula Archiver

The archiving module of the Granula framework automates the process of converting performance logs into a self-contained, self-explanatory, queryable performance archive. The execution of the archiving process is purely based on the performance model of a BDP system, while independent from the BDP system itself. Such design liberates performance analysts from repetitive, tedious work of processing performance data, and therefore be able to focus mostly on the performance modelling and statistical analysis.

Granula Visualizer

The visualization module of the Granula framework facilitates interactive content browsing of Granula performance archives, which allows quick navigation to the interesting part of the BDP job, and complete traceability of every piece of performance information to its sources. The visualization process is rendered purely based on the performance archive of a BDP system, thus independent from the BDP system itself.
